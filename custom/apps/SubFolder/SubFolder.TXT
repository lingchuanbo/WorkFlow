概要
  このプログラムは マイコンピューターから存在する全てのフォルダーをメニューに
表示しそれを追って深いフォルダーでも一回で開く事が出来る SbFolder の 反応と
機能拡張の為に有る部分を常駐させる為に 別の SubFoldJO.EXE に分けた物です。
機能的には SbFolder と同様にコマンドラインによって途中のフォルダーからも始める
事が出来ます。大きな違いは Win XP 以前ならフォルダー/ファイルの右クリックでエクスプローラーが
出す 右コンテキストメニューを直接出す事が出来る事です。
  テキストは SubFolder.EXE について書いてありますが実質は SubFoldJO.EXE の方に
有り SubFolder.EXE を SubFoldJO.EXE と読み替えても同じ動作です。SubFolder.EXE は
SubFoldJO.EXE が常駐していなければ此をコマンドラインと共に立ち上げ 常駐していれば
コマンドラインを SubFoldJO.EXE に送るだけと言う動作の中身は殆ど何もない物です。

特徴
  マイコンピューターから存在する全てのフォルダーをメニューを追うことで開く事
が出来ます。コマンドラインによって途中のフォルダーからも始めることが出来る
ので送るなどにショートカットをおいておけばそのフォルダーの下のフォルダーから
メニューで探しながら開く事が出来ます。
  SubFolder.INF の右コンテキストメニューから インストールをすればフォルダーの
右コンテキストメニューに Sub Folder (B) メニューが加わりますのでそのフォルダー
から始める事が出来るようになります。
マウスの右クリックでメニューの経歴を 4 個まで貯めることが出来るので 一回の立ち
上げで 最大 5 個のフォルダーを 同時に開く事が出来ます。選択した物が ファイルなら
関連付けプログラムで開く事も出来ます。
フォルダー ファイルを 開く為の一回きりの起動で常駐等はしません。
リストテキストを読み込む事によって良く使用するフォルダーやファイルもすぐに
立ち上げる事もできます。


●動作環境
  Win 98  NT 4.0  2000 XP Vista 7 8 10 特別なランタイムは必要ありません。

●インストール方法
適当な所に SubFolder.EXE SubFoldJO.EXE SubFolder.HLP をコピーして 適当な所に( ディスクトップ等 )
にショートカットを作って立ち上げればドライブ表示のメニューから始めますが フォル
ダーの途中から始める場合は下記の方法を採って下さい。
・マニアルでのインストール
* SubFolder.EXE SubFoldJO.EXE SubFolder.HLP を適当なフォルダーにおいて下さい。
C:\WINDOWS\SendTo フォルダーに  SubFolder.EXE の Short Cut を作り
適当に名前を付け替えてください。( 変えなくても良い ) 送るメニューから使えます。
あと適当な所に( ディスクトップ スタートメニュー 等にShort Cut を作って下さい。)
又 ディスクトップにショートカットを作ってそれにフォルダーをドロップする事でも
使用できます。
  アンインストールはただ SubFolder.EXE と同じフォルダーに有る SubFoldJO.EXE SubFolder.HLP 又 各ユーザーの
 Application Data フォルダーに有るSubFoldrVF.txt とSubFoldrPG.txt ショートカットを削除して下さい。
他には何もしていません。

* SubFolder.INF ( セットアップ情報 ) を使用したインストール
  SubFolder.INF を右クリックして出るコンテキストメニューからインストールを選んで
下さい。
  SubFolder.exe → 現在ファイルの置かれている有るフォルダー
  SubFoldJO.exe → 現在ファイルの置かれている有るフォルダー
  SubFolder.chm → 現在ファイルの置かれている有るフォルダー
  SubFolder.INF → C:\WINDOWS\INF\SubFolder.INF
  レジストリーにフォルダーの右クリックのコンテキストメニューに Sub Folder (B)
の項目を加えます。送るから選択するより的確で早いでしょう。
ディスクトップ 等の ショートカットは適宜マニュアルで作って下さい。

  アンインストールはアプリケーションの追加と削除から Sub Folder (Remove)を
選んで下さい。SubFolder.INF レジストリー 関連は全て消す事が出来ます。
インストールフォルダーに有るファイル は削除して下さい。
後 各ユーザーの Application Data フォルダーに残る SubFoldVF.TXT SubFoldrRC.txz は マニュアルで削除
して下さい。

* 注意 ルートからのパスが全て 8文字以内のフォルダーパスでないと旨くいかないかもしれません。

レジストリ情報は
  HKCR,"Folder\shell\SubFolder",,,"SubFolder"
  HKCR,"Folder\shell\SubFolder\command",,,"%10%\SubFolder.EXE %1"

  HKLM,"Software\Microsoft\Windows\CurrentVersion\Uninstall" 
  ApliTitle = "SubFolder"
  に書き込んでフォルダーの右クリックメニューに登録します。

【アップデート】
 新しい SubFolder.EXE SubFoldJO.EXE を インストール フォルダーの SubFolder.EXE SubFoldJO.EXE

キーで常駐を解除するか Windows を立ち上げた直後に 上書きして下さい。
もちろん タスク マネージャ から プロセスの終了でも問題は有りません。

●使用方法
  ショートカットなどからコマンドラインなしで単に立ち上げればドライブ表示の
メニューから始めます。コマンドラインにフォルダー名を入れればそのフォルダーの
下のフォルダーをメニューにして始めます。
  フォルダーの右クリックメニューから (送る) →Sub Folder 左クリックで
そのフォルダーの下のフォルダーのメニューが出ます。
下のフォルダーが無い時 又は フォルダー名の下に出る数はそのフォルダーの下の
通常のファイルの数です。
スタートメニューにショートカットを作り
  ・・・\SubFolder.EXE C:\Program Files
と入れておけば C:\Program Files 以下のフォルダーのメニュー表示から始まります。
ネットワークにも対応しています。( \\名前\共有名\... と書いて下さい。)
全てのメニューはサブメニュー形式で示されますがフォルダー名の所を 左 若しくは
右 クリックして下さい。( ファイルの数の所でも良い ) 右クリック ではそのフォル
ダーを経歴として記憶します。( 記憶すると質問時に出るビープ音が鳴ります。)
フォルダーの記憶は 4 個までで それ以上右クリックをすると古い物から消されます。
左クリック ではそのフォルダーを開きます。
  マウスボタンをクリック する時に Shift キーを押していると フォルダーを開かずに
クリックした フォルダー ファイルの Full Path をクリップボードにコピーします。
  フォルダーを記憶又は開く時に Ctrl を押しながらクリックするとそのフォルダーの下の
全てのフォルダーを開きます。全てのフォルダーを開いている時に途中で終了したい時には
Shift or ESC を押して下さい。そこまでで終了します。
したがって マウスボタンをクリック する時に Shift + Ctrl だと フォルダーならその下にある
全ての フォルダー ファイル名をクリップボードにコピーします。ドライブならそのドライブに
有る全てのフォルダー ファイル名をクリップボードにコピーする事になります。
  あまりにたくさんの メニューが出て上のフォルダーに( 隠れてしまったメニューに )戻り
たい時は Control か BS か Home か ← キーを押して下さい上のメニューが消えて下のメニュー
にアクセス出来ます。
  ファイル/フォルダーを記憶又は開く時に 無変換 / Ｆ２ キーを押しながらクリックすると選択
ファイル/フォルダーに対するエクスプローラーの右コンテキストメニューを出します。
  ファイル/フォルダーを記憶又は開く時に F3 キーを押しながらクリックすると *.EXE *.COM
 *.LNK *>PIF ならそれを開く前に コマンドライン指定の *.EXE *.COM で無い時には それを
開く プログラムを編集するダイアログを出しますので 必要なら此に入れて下さい。ダイアログは
OK なら実行ですが 右上の X は キャンセルで その物自体をスキップします。
  フォルダーを記憶又は開く時に F4 キーを押しながらクリックすると 左に TreeView の
付いた ( 所謂 エクスプローラーで開くと言うのでしょうか ) 状態で開きます。コマンドラインに /+E を
指定した時には TreeView 無しで開きます。
OK を押す時に Ctrl を押しているとコンボボックスに出る プログラムリストを保存している
SubFoldrPG.TXT を編集できます。此に 良く使う プログラム を書いて下さい。パスの通っている
プログラム ( ex. NOTEPAD 等 )なら そのままで OK ですが Full Path で書いた方が確実です。
プログラムを編集する時に....EXE の後に 選択したファイル名の前後に コマンドを加える事が
出来ます。
....EXE;1234;5678 という風に加えて下さい。1234 が 選択したファイル名の前に 5678 が 選択した
ファイル名の後ろに付きます。;1234 なら前だけ ;;5678 なら後ろだけです。
( 前の 1234 は バッファの関係で 24 文字を越える物は無視します。)
....EXE;";" は "" で括った "選択したファイル名" を コマンドラインに送る事になります。
勿論 最初から SubFoldPG.TXT に書いて選択した時も有効です。
  ファイル/フォルダーを記憶又は開く時に F3 キーを押しながらクリックすると関連付けの無い時に
出る " ファイルを開くアプリケーションの選択 " を出しますのでこちらからでも立ち上げる事が
出来ます。SubFoldrPG.TXT に用意していない物で開く時は冗長ですが此も使えるしょう。
  フォルダーでない通常のファイルの数を 表示している ○○ 個_ と書いてあるメニューを
選択すると(Space キー 押しでも) そのフォルダーに有るファイル名を表示します。
此をクリックするとそのファイルが関連付けられている物なら直接開く事が出来ます。
関連付けられていない時はそのファイルを選択してファイルの有るフォルダーを開きます。
関連付けられているファイルを選択してファイルの有るフォルダーを開く時は Ctrl を
押しながら選択して下さい。
  あまりに フォルダー ファイルの多いフォルダーでは途中までしか表示できずに最後が
- 残り00 個_- となりますがこのフォルダーを開きたい時には此を選択するか Space を押して下さい。
この  - 残り00 個_- と出た時に Ctrl で次の物をBS で前の物を Home キーで最初から表示出来ます。
但しこの Ctrl BS Home キーの方法は フォルダー深さが 浅い 他のポップアップメニューを作らなかった
又は 深いフォルダーで  - 残り00 個_-  が出なかった 場合に限ります。
  選択したフォルダー ファイルが有る時に ALTキー を押すと 又は マウスの左ボタンを
0.7 秒以上長く押す事でも 選択したフォルダー 又はファイルが有るフォルダーから 再び
SubFolder を立ち上げます。あまりに深いフォルダーを追いすぎた時や SubFoldvf.TXTの項目
を選んだ時などに活用して下さい。
  ドライブからでない途中から始めた時は最後のメニューに 一段上に Tab が加わります
此を選択するか Tab キーを押す事で1つ上のフォルダーをメニューにして再開します。
 一段上に Tab を 0.7 秒以上長く押すか 此が選択されている時に Alt キー を押すと 一番最初の
ドライブのメニューから始めます。
  フォルダー を開く為 左クリック をした時に 警告音がする場合は逆に戻ったりして
パスが狂っている時です。( この様な時はその先のメニューも出ません。一度メニュー
の選択を上下して選択を変えて下さい。先のメニューが出るようにならば OK です。)
記憶する為の 右クリックをした時に 警告音がする場合は記憶するパスが変わってい
ない時です。( 単に一度は動いたかを見ているだけで内容を比較している訳ではない )
フォルダーを開く時は古い順に開きます。( 最後に左クリックした時のフォルダーが
一番上に来ます。) メニューの所でない箇所をクリックして終了した時は記憶したフォルダー
があっても開きません。( 全てキャンセルです。記憶したフォルダーは開きたい時には SHIFT
を押しながら終了して下さい )
  バージョン情報 を右クリック ( or 左クリック + Shift ) した時には マイ コンピュータを開きます。
バージョン情報 の左クリックを 0.7 秒以上押すか Alt キーで バージョン情報 の下に出るメニューから始めます。
バージョン情報 を 左右クリックする時に Contrl を押しているか 右クリックで 0.7 秒以上長く押すかすると
バージョン情報 の下のメニューの元になっている ( 通常はSubFoldrVF.txt ) ファイルを編集出来ます。
此で バージョン情報 の下に  出る 良く使うフォルダーやファイルを編集して下さい。
バージョン情報の上に経歴ファイル と言うメニューが出ます。SubFolder で開いた フォルダー ファイルを 
使った順に 40 個 メニューにして出します。単なるメニューでポップアップで追う事は出来ません。
此のメニューの数は最大 40個 文字数は 4 文字以上 72文字 以下です。従ってC:\ D:\ とかは記憶されず 又あまりに
深かったり長かったりするファイル名は記憶されません。此は 各ユーザーの Application Data フォルダーに
 SbFolderRC.txz として記憶されます。この機能が不必要な時にはコマンドラインに /-R を書いて下さい。
  フォルダーを記憶しなかった場合は 左クリックした時のフォルダーしか開きません。
メニューの出ている時に F1 キーを押すと使用法の ヘルプが出ます。
  メニュー作成に時間のかかる数の多いネットワークや遅いドライブのメニュー等に入って
しまった時に Shift キーを押すと 此を途中で キャンセル出来ます。もちろん出たメニューは
途中までで全てでは有りません。

【コマンドライン】
  コマンドライン に 表示開始位置指定を入れることが出来ます。
・・・\SubFolder.EXE [フォルダー名 /-(AD) /-NET /-VF /+E ] [/20,200] [/.DLL.WMA.FM .OCX or /.*][*.TXT]
フォルダー名： 此のフォルダー以下をメニューにして始めます。
  フォルダー名 に \\ : ネットワーク から始めます。
  最初のドライブから始める時に コマンドライン によって省くドライブを指定する事が
出来ます。
/-(AD)： で A D ドライブを
/-NET ： でネットワークを省いて始めます。
  時間のかかる物や要らない ドライブ ネットワークを表示させたくない時に指定して下さい。
＊ /(AD) /-NET は最初のドライブメニューの時に有効です。
/-VF ：編集した SubFold[UseName]VF.TXT の内容から始めます。
/+E  ：フォルダーを開く時に Folder Tree ( TreeView )付きで開きます。
/-H ：隠しフォルダー 隠しファイルはメニューの対象から外します。
/-R ：経歴ファイル のメニュー機能を無効にして上がります。( 内部的な記憶もしません )
/-L ：フォルダー名が 24文字を超える物は メニュー表示は 短い名前にして ( DOS 規則の名前 ) 表示メニューの
  幅が大きくなりすぎて見にくくなるのを防ぎます。ただし フォルダーによってはDOS 規則の名前に変換出来ない
  物もあるらしく全てが短くなる訳でもないようです。
/-M ：CD-ROM や リムーバブルドライブ ネットワーク等のドライブの有無は確認しますがメディアやアイテムの
  確認はしません。メディアやアイテムの確認をしないので立ち上がりにこの為に待たされる事は有りません。
  後で メディアの入っている物は メニューを出す事は出来ます。但し ボリューム名は 取得されません。
   立ち上げ時の 無変換キーと排他的トグル動作となります。
/-(AD),/-NET と /-VF と フォルダー名 は排他で先に出た方が優先します。
..\xoxo.TXT：( Full Path ) で バージョン情報 の下に出る 良く使う フォルダーやファイル の
  SubFoldrVF.TXT と同じように書いたテキストファイルを指定すると此の内容のメニューから
  始める事が出来ます。簡易ランチャーとして使用する時に使用して下さい。
  此の Full Path テキストの前に ; を入れれば此が SubFold[UseName]VF.TXT の代わりとなり最初から始めます。
 
/20,200：表示位置指定です。
  , の前の数字が左から 後の数字がが上からの位置にマウスカーソルを置いて立ち上がります。
  ランチャー スタートメニュー ディスクトップ等から常に同じ位置で立ち上がって欲しいと時には
  ショートカットのプロパティーのリンク先に
  ・・・\SubFolder.EXE /20,200 等と入れておけば常に同じ位置からスタートします。
  ( 上記では ( 20, 200 ) にマウスカーソルを置いて全てのドライブから始めます。)
  /20 では 横位置だけ /,200 では縦の位置だけの指定になります。
/20'200：表示位置指定ですがマウスカーソルは動かさずにメニューをその位置から出します。

   /.DLL.WMA.FM .OCX：対照除外フィルターです。 *.DLL *.WMA *.FM *.OCX はファイルの表示
  ( 数のカウント ) にも入れないでメニューを出します。
  同種ファイルが非常に多い時表示の必要の無いファイル排除等に使用して下さい。
  拡張子が 2文字の場合はスペースを 1つ 拡張子が 1文字の場合はスペースを 2つ入れて下さい。
  拡張子が HTML 等の 4文字の場合は最初の 3文字 HTM だけを入れて下さい。
  /.* で全ての通常のファイル( フォルダー以外 ) は数だけでメニューにしなくなります。
  フォルダー表示だけで良い時は此を指定すると表示速度が速くなり 画面もすっきりします。
  /R で常駐している SubFolder.EXE を終了させます。
  

【立ち上げ時のキーキーオプション】
下記 キーを押しながら立ち上げる事で
無変換 : コマンドラインの フォルダー名 の一段上のフォルダーから始めます。
  この時 Shift を同時に押していると 一番上( ドライブ ) から始めます。
  一番上( ドライブ ) から始める条件の時に押されていると CD-ROM ドライブ リムーバブルドライブ
  のディスクの確認をしない様にします。準備が出来ていない ( 入れた直後等 ) ドライブが有る時は
  無変換キーを押す事で待たずに立ち上げる事が出来ます。
Shift : コマンドライン の /-VF と同様に 編集した SubFold[UseName]VF.TXT の内容から始めます。
  上記 二つ共に 位置 /20,200 や 除外 /.DLL.WMA.FM .OCX or /.* のオプションは有効です。
Ctrl  : 除外  /.DLL.WMA.FM .OCX or /.* が有る時は 此を無くし 無い時は /.* を付けた
  のと同じ動作をします。 位置 /20,200 オプションは有効です。

●使用に関して
  このソフトは 正常な動作 バグ等が無い事 を期待されて作られてはいますが全ての環境での正常動作の
保証はいたしかねます。又 このソフトの正常動作 不良動作によって引き起こされた何らかの被害等の保証も
いたしかねます。ただし 仕様上の御意見 動作不良のレポート等は 歓迎いたします。


●制限事項
   一番深い扱えるフォルダーはルートから 32 個までです。
一つの フォルダー名 ファイル名の長さ は 半角で 215 文字までです。但し一番最初に
立ち上げる時にはメニューが画面の幅より長くなるとメニューが消えて見えなくなって
しまう様です。立ち上がって二段目以降のメニューで表示する時には大丈夫な様です。
( それ程 深いフォルダー 長い名前は Windows の Full Path 名が 259 までなので殆ど
あり得ないと思いますが念の為 )
  一つのフォルダーの下にフォルダーがたくさん有る時には横に並べて表示しますが余り
にたくさん有りすぎる時には画面から溢れて選択できなくなるかも知れません。
  深いフォルダーをどんどん開いていく時には問題は無いのですがそこからもどる時
には出来るならゆっくりメニューを消しながらさかのぼるとメニューと内部で保持して
いる パスの整合性が取れるのですが あまり早くさかのぼるとメニューと内部で保持し
ている パスが違ってしまう事が有るかも知れません。特にメニューの外側からルートの
方に戻った時にはパスは違い易くなります。( ひょっとすると一般保護違反が出るかも
しれません。Ver 2.03 からは殆ど関係なしで制限は無くなっていると思いますが。)
この為に有る程度のタイムディレイを入れていますので多少一カ所で待ってみて下さい。
メニューが実際と合わない時は ( メニューをクリックしても 警告音がしてフォルダー
がその時は開かない様にはしています。 ) 一度メニューの選択を上下して選択を変えて
みて下さい。フォルダーの内部パスが実際と合い開く事が出来るようになります。
( 此はメニューの書き換えはメニューの選択が変わった時にしか行えない為です。)
フロッピーディスクと ネットワークについては読み込むのに時間がかかるので選択した時
にしかチェックない仕様にしています。この為 フロッピーディスクについては入ってなく
てもいても最初は有効の表示になっていますが選択することで正しい表示になります。
フォルダーでない通常のファイルを メニューにする数は  画面のサイズによってメニュー
を縦に並べる数の 3 〜 6 倍にしています。画面が広いほど多数のメニューを表示します。
( メニューで画面がいっぱいになってしまう程のファイルの有るフォルダーがある為です。)
それ以上ファイルの有る時は フォルダーを開いて確認して下さい。
  バージョン情報の下に出る自分で編集したフォルダーからメニューを出している時に此の
メニューを辿って浅い方に戻った時に最後の自分で編集したフォルダーは此処でフォルダー
深さとメニュー深さのギャップがあるので場合によっては正確には開か無いかも知れません。

無変換 / F2 で出るエクスプローラーの右コンテキストメニューの中の Sub Folderは
働きません。( SubFolder の 二重起動は認めていません。)


●注意事項
  フォルダーでない通常のファイルのメニューを選択する時には良くエクステンションを確認して
選択して下さい。直接プログラムが立ち上がる事がありますので何が起こるか良く考えてから
選択して下さい。( Ex. *.BAT / *.INI / *.INF / *.REG 等 )

  フォルダーを記憶又は開く時に Ctrl を押しながらクリックするとそのフォルダーの下の全ての
フォルダーを開きますが あまり数の多いフォルダーではシステムリソースが足りなくなる事が
あります。( 一応 システムリソース を見て 途中で中止するようにはしています。)
Ctrl + Shift 両方押しているとそのフォルダーの下の全ての フォルダー / ファイルの Full
Path をクリップボードにコピーします。

SubFoldJO.exe の製品名に Reverse M Botton と書いてある 左利き用の SubFoldJO.exe をご使用の
方は 此の テキストと ヘルプファイルの マウスボタンの左右を入れ替えて読んで下さい。

ファイル/フォルダーを記憶又は開く時に F3 キーを押しながらクリックして出る コマンドライン
/ プログラム 指定のダイアログはどんな種類のファイル/フォルダーの場合でも出ますので意味の
あるコマンドライン / プログラム を入れて選択して下さい。


● バージョン情報 の下に出る 良く使う フォルダーやファイル の SubFold[UseName]VF.TXT について
  各ユーザーの Application Data フォルダーに バージョン情報 を選択した時に 無ければ SubFoldrVF.txtをに
作ります。単なる テキスト形式の フォルダーやファイルを一行に書いたテキストファイルです。
編集するには Version情報 をクリックする時に Contrl を押しているか 右クリックで0.7 秒以上長く押して下さい。
テキストファイルに関係付けられたエディターで開きます。
 1つのメニュー( ファイル名やフォルダー名 )を一行に書いて下さい。( この様な フォルダーパスやファイルの
フルパスを書くときには 同じホームページにある ClipName / ClipSaver が便利です。)
勿論 直接 エディターで 各ユーザーの Application Data フォルダーに有る SubFoldrVF.txt を編集しても同じ
事です。
ファイル名やフォルダー名 の前に

@スタートメニュー  	←( 行の最初に @を入れる )
C:\Documents and Settings\All Users\スタート メニュー	←此は常にフルパスで
   
と書けばメニューのタイトルは スタートメニューで開いていくメニューは
  C:\Documents and Settings\All Users\スタート メニュー
になります。あまりに深いフォルダーや解りにくいフォルダー名等の時に使用すると分かり易くなるかと思います。
ネットワークコンピューター は \\コンピューター名 と書けば 此を選択する事で開く事が出来ます。
ローカルドライブについては準備が出来ていないとメニューに加えませんがネットワークコンピューターと
A B ドライブ ( フロッピー ) については常に表示します。( メニューを作るのに時間がかかってしまいメニューの
反応が遅くなるのを避けています。) 後 メニューの深さの整合性の為 C:\ 等のドライブは認めません。
! の行頭が有れば メニューのコラムを変えます。
　　数が多く高さが高くなりすぎたりメニューの種類によってコラムを分けたい時に使用して下さい。
- の行頭が有れそこに セパレーターを書きます。
　　纏まった区切りに使用すると分かりやすくなります。

行頭が 上記 ! - @ 以外の A より小さい文字から始まっている行は何もしません。( よく言えばコメント行です。) 
  表示数についての制限は有りませんがあまり多いと画面に入らなくなります。ソートもせず単に
SubFoldrVF.txtに書いてある順にメニューにするだけですので気に入る順序に編集して下さい。
  もしあまりに長いフォルダー名の場合は DOS Name を使うと短くできます。
   C:\Program Files\Windows Media Player\Visualizations   は
   C:\PROGRA~1\WINDOW~1\VISUAL~1    と同じ事です。( 此も ClipName を使用しました。)
コマンドラインに指定する テキストファイルも此と同様にして下さい。
SubFolder に上記テキストと同じように書いた物を送る事によって此の内容から始める事が出来ますので何らかの
ランチャーとして使えるかもしれません。

● F3 を押したときに出る ダイアログボックス の プログラム選択用コンボボックスのリスト保存用の
  保存用のSubFoldrPG.txt について。
各ユーザーの Application Data フォルダーに SubFoldrPG.txt を無ければ作ります。
コマンドライン プログラム 編集 選択 ダイアログの OK を押す時に Ctrl を押していると此を編集出来ます。
勿論 直接 エディターで 各ユーザーの Application Data フォルダーに有る SubFoldrPG.txt を編集しても
同じ事です。
SubFolder での選択が *.EXE *.COM 等の実行ファイルなら ダイアログは コマンドラインの付加 となり単に
これらの実行ファイルに渡す コマンドラインを入れて下さい。
選択 が上記以外の ファイル や フォルダーの時は ダイアログは 開く Program となります。
開くプログラム指定に プログラムだけなら プログラムに選択したファイル名 or フォルダー名を渡します。
開くプログラム指定の時に ***.EXE;1234;5678 という風に加えると 1234 を 選択したファイル名の前に 5678 を
後ろに付けて 指定された開くプログラムを実行します。;1234 なら前だけ ;;5678 なら後ろだけとなります。
プログラム 1234 選択したファイル名 5678 と言う のを名前を指定して実行するのと同じです。
***.EXE;";" は "選択したファイル名" と "" で括る事になります。
一行ずつ プログラム名 NOTEPAD やら WINWORD ( パスの通っているプログラム ) と書いて下さい。
パスの通っていない物は フルパスで書いて下さい。順序は書いてある通りに コンボボックスのリストに
並べますので 良く使う プログラムを自分なりに書いて下さい。
ダイアログ 立ち上げの時にコンボボックスに代入するので選択して入力出来る様になります。

●Reverse Mouse Button
  SubFoldJO.exe は直にマウスボタンの左右を読み取っています。したがって 左手でマウスを使用する方で
  常駐型左右入れ替えソフトではボタンが反転しない事が有ります。
  その様な時には Reverse Mouse Button と製品名に出る SubFoldJO.exe をダウンロードして使用して下さい。
